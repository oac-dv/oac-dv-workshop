Exercise 1 of 5

Build the Dataset Foundation

Connect your five data tables and create one trusted dataset. Everything else depends on getting this right.

What you're building: A single dataset called Customer Insurance that joins Policies, Customer Premiums, Insurance Claims, Drivers, and Cars Insured with Policies as the driving table. This is the data layer that all three canvases share.
0 / 9 steps done 0%
Part A — Upload the data file ~8 min
  • 1
    From the OAC Home page, click the Create button in the top right corner
  • 2
    Click Dataset from the dropdown menu
  • 3
    Click Drop data file here or click to browse and select the file: Car Insurance - Only Required Data.xlsx
  • 4
    When OAC shows the sheet list, select all sheets (Policies, Customer Premiums, Insurance Claims, Drivers, Cars Insured) — each sheet becomes a separate table in your dataset
Why select all sheets? Each Excel tab holds a different part of the customer story. Selecting all of them lets OAC connect them together, which is what makes the analysis powerful.
Part B — Set ID fields as Attributes ~5 min
Do this before saving. ID fields (Customer ID, customer_policy_id, Claim ID, Driver ID, Driver Assigned) look like numbers, but they are labels — not values to add up. If you skip this step, your counts will be wrong.
  • 5
    For each ID column (Customer ID, customer_policy_id, Claim ID, Driver ID, Driver Assigned, etc.), click the column header and look for the data type icon — change it from Measure (#) to Attribute (A)
  • 6
    Confirm the change for all ID-style columns before moving on
Part C — Configure the joins ~7 min
What is a join? A join tells OAC how to connect two tables — "when customer_policy_id in Table A matches customer_policy_id in Table B, treat those rows as the same policy/customer." Without joins, the tables are just separate lists.
  • 7
    Click the Diagram tab to see the join view. Make Policies the driving table and set Preserve Grain to the Policies table.
  • 8
    Configure every connection as a Left join. Use the join map below, including both join conditions for Drivers → Cars Insured.
  • 9
    Click Save, name the dataset Customer Insurance, then click Create Workbook.
JoinJoin TypeJoin Conditions
Policies to Customer PremiumsLeft joinPolicies.customer_policy_id = Customer Premiums.customer_policy_id
Policies to Insurance ClaimsLeft joinPolicies.customer_policy_id = Insurance Claims.customer_policy_id
Insurance Claims to DriversLeft joinInsurance Claims.customer_policy_id = Drivers.customer_policy_id
Drivers to Cars InsuredLeft joinDrivers.customer_policy_id = Cars Insured.customer_policy_id
Drivers.Driver ID = Cars Insured.Driver Assigned
Quick check: After saving, your dataset should show all five table names in the Data panel inside the workbook. If any table is missing, go back to the dataset editor.

Exercise 2 of 5

Create the Six Decision Metrics

Translate raw data into business language. Each formula answers a specific business question — understand the "why" before typing the expression.

Where to create calculations: In your workbook, right-click on the My Calculations section in the Data panel (left side), then click Add Calculation. Type the formula, name it, and save. You'll do this six times.
0 / 6 steps done 0%

Calculation 1 — Policy Count

Business question it answers: How many unique customers do we have in total?
FormulaCustomer base
COUNT(DISTINCT "Customer ID")
Why DISTINCT? After joining tables, one customer can appear on many rows (one per claim, one per premium period). COUNT DISTINCT collapses them back to one count per customer.
  • 1
    Create calculation named Policy Count with the formula above. Save it.

Calculation 2 — Claims Count

Business question it answers: How many claim events happened? (Not dollar value — number of incidents.)
FormulaClaim volume
COUNT(DISTINCT "Claim ID")
  • 2
    Create calculation named Claims Count with the formula above. Save it.

Calculation 3 — Churned Policy Count

Business question it answers: How many customers have already left?
FormulaChurn count
FILTER(COUNT("End of Policy Renewal") USING "Renewal Status" = 'churned')
What FILTER does: It counts renewal records only where the customer churned — it ignores all other rows. This gives you a clean count regardless of how many rows the join created.
  • 3
    Create calculation named Churned Policy Count with the formula above. Save it.

Calculation 4 — Churn Rate

Business question it answers: What percentage of our customers churned? A count of 50 churned policies means very different things in a base of 200 versus 2,000.
FormulaRetention risk
"Churned Policy Count" / "Policy Count"
Format tip: After creating this calculation, right-click it and set the number format to Percentage with 1 decimal place.
  • 4
    Create calculation named Churn Rate with the formula above. Format as percentage. Save it.

Calculation 5 — Loss Ratio

Business question it answers: For every $1 of premium collected, how much did the company pay back in claims? A loss ratio of 0.60 means the company paid out 60 cents of every dollar it earned.
FormulaProfitability signal
"Insurance Paid" / "Premium Amount"
Why this matters for retention: A customer with high premium but also high claims may not be as valuable as they look. Loss Ratio prevents us from prioritizing costly customers by mistake.
  • 5
    Create calculation named Loss Ratio with the formula above. Format as percentage. Save it.

Calculation 6 — Customer Value Score

Business question it answers: How much net value has this customer generated? Simple signal: premium collected minus claims paid.
FormulaPriority signal
"Premium Amount" - "Insurance Paid"
How you'll use it: This score goes on the X-axis of your retention scatterplot in Exercise 5. Higher score = more net value to the business = stronger case for prioritized retention effort. Some source screenshots label this same priority idea as Customer LTV; in this 3-hour handout, use the simplified Customer Value Score formula above.
  • 6
    Create calculation named Customer Value Score with the formula above. Save it.
Quick check — all 6 saved? In the Data panel under My Calculations, you should see: Policy Count, Claims Count, Churned Policy Count, Churn Rate, Loss Ratio, Customer Value Score. If any show a zero or error, check that your ID fields are set to Attribute.

Business Overview Canvas

This canvas answers: What is the current state of the business? Build the big picture before zooming into retention.

How to add a new canvas: At the bottom of your workbook, click the + button next to the canvas tabs. Rename it Business Overview by double-clicking the tab name.
0 / 7 steps done 0%
How to create a visualization: Double-click a field in the Data panel to add it to the canvas. Or drag a field from the Data panel and drop it on the canvas. OAC will suggest a chart type — you can change it using the chart-type picker in the top-left of the visualization.

Build these 7 visuals in order

Tile: Policy CountShows total number of customers. Drag Policy Count → change to Tile type.
Tile: Claims CountShows total claim events. Same approach as above.
Tile: Churned Policy CountThe raw churn number.
Tile: Churn RateFormatted as %. This is your key business health signal.
Bar/Line: Claim Amount by Date of IncidentShows claim activity trend over time. Are claims going up or down?
Map: Claim Amount by StateGeographic view. Drag State → Map, then add Claim Amount as the color measure.
(Optional) Line: Churn Rate by Renewal MonthOnly if you have time — shows churn trend over renewal periods.
Stuck on the map? If OAC doesn't recognize your State field as a geographic type, right-click the State column in the Data panel → select Location Details → choose State/Province and set the country to USA.

Reading the scatterplot — the four zones

✓ Call First

High Customer Value Score + Low Loss Ratio. Good net value and manageable claim cost. Strongest retention ROI.

⚠ Review Before Acting

High Value Score + High Loss Ratio. Potentially valuable but costly. Understand claim history before making an offer.

→ Consider Later

Low Value Score + Low Loss Ratio. Stable but low return. Not urgent for outreach.

✗ Lowest Priority

Low Value Score + High Loss Ratio. Low value and high cost. Not where to focus retention effort.
